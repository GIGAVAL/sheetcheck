# CLAUDE.md

Guidance for Claude Code when working in this repository.

## Version control workflow

As you do work, commit to Git and push to GitHub regularly so we never lose the status or work we've done.

- **Commit often.** After each meaningful, self-contained unit of work (a fix, a feature, a refactor step), make a commit rather than batching many unrelated changes into one.
- **Push regularly.** Push commits to GitHub `origin` promptly after committing so work is backed up remotely and the remote reflects current status.
- **Write clean commit messages.** Use a concise, imperative summary line (e.g. "Add zero-install browser demo") that describes what the commit does. Add a body when the change needs context.
- **Keep the tree in a known-good state.** Prefer committing working increments; note in the message if something is intentionally incomplete.
- **Don't lose work.** If in doubt, commit and push — it's cheaper to have an extra commit than to lose progress.
